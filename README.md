Sistema de Gestión de Obras Urbanas

Proyecto desarrollado en Python para gestionar obras públicas de la Ciudad de Buenos Aires, incluyendo procesamiento de datos, persistencia en base de datos y simulación del ciclo de vida de una obra.

Objetivo:
Desarrollar una aplicación orientada a objetos capaz de:
- Procesar datasets reales en formato CSV
- Limpiar y normalizar datos
- Persistir información en base de datos SQLite mediante ORM
- Simular el flujo completo de gestión de obras

Tecnologías utilizadas:
- Python
- Pandas
- NumPy
- Peewee (ORM)
- SQLite

Funcionalidades principales:
- Importación de datos desde CSV
- Limpieza y transformación de datos
- Modelado de datos con ORM (Peewee)
- Persistencia en base de datos SQLite
- Gestión del ciclo de vida de obras:
  - Creación de proyectos
  - Contratación
  - Adjudicación
  - Ejecución
  - Finalización o rescisión

Análisis e indicadores:
El sistema permite obtener:
- Cantidad de obras por etapa
- Inversión total por tipo de obra
- Obras finalizadas en determinado plazo
- Distribución por barrios y áreas responsables
